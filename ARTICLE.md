# בניתי לעצמי מנוע חיפוש עבודה עם AI — והנה הפרומפט שיבנה לך אחד תוך 10 דקות

לפני שבועיים התחלתי לחפש עבודה. כמו כל מי שעוסק ב-RevOps, השאלה הראשונה שלי לא הייתה "איפה אני מגיש" אלא **"איך אני בונה מערכת שתעשה את זה בשבילי"**.

אז בניתי אחת. סוכן AI שסורק משרות 3 פעמים ביום, מנקד כל אחת לפי הפרופיל שלי, ומעדכן טראקר חי — אוטומטית. תוך 10 ימים: 25 הגשות, 5 ראיונות, 3 שעברתי.

הדבר הכי טוב? **אתה יכול לבנות לעצמך אחד תוך 10 דקות** — בלי לכתוב שורת קוד. רק להדביק פרומפט אחד ל-Claude Code.

---

## מה המערכת עושה

- 🔍 **מוצאת משרות** — Claude סורק LinkedIn, Greenhouse, Lever, Ashby ומחזיר לינקים ישירים
- 🎯 **מנקדת התאמה 1–10** — לפי הפרופיל, המיקום והעדפות שלך
- 📊 **טראקר חי** — סטטוסים, סבבי ראיונות, הערות, אנליטיקה — באתר אמיתי שמתעדכן לבד
- 🧠 **לומדת ממך** — מסמן משרה "לא רלוונטי"? היא לא תוצע שוב
- ✉️ **מייצרת מכתבים, קו״ח מותאמים והכנה לראיון** — בלחיצה

הכל רץ על ה-GitHub שלך, עם הדאטה שלך, בחינם (חוץ מ-~$5–10 לחודש על ה-API).

## איך בונים — 4 צעדים

### צעד 1: פותחים חשבון Anthropic API
נכנסים ל-`console.anthropic.com`, מוסיפים $10 קרדיט, ומייצרים API key. (זה לא Claude Pro — זה ה-API, pay-as-you-go.)

### צעד 2: מדביקים את הפרומפט הזה ל-Claude Code

```
בנה לי מנוע חיפוש עבודה אוטומטי מבוסס Claude, פרוס על GitHub Pages.

## שלב ראשון — אונבורדינג: שאל אותי קודם
1. אילו תפקידים אני מחפש?
2. איפה אני מוכן לעבוד? (מיקום / רימוט)
3. מה הניסיון והחוזקות שלי?
4. בקש שאעלה קורות חיים (PDF/TXT)
מהתשובות צור config.json + profile.md.

## ארכיטקטורה
run_pipeline.py:
  - search_jobs()  → Claude Opus + web_search → מוצא משרות פתוחות אמיתיות
  - ats_scraper    → Greenhouse/Lever/Ashby API ציבורי (חברות שאבחר)
  - score_jobs()   → Claude Haiku → מנקד 1-10 לפי הפרופיל
  - filter         → ציון ≥4 + מיקום מתאים + עלתה ב-30 הימים האחרונים
  - save           → scored_jobs.json (מקור אמת יחיד)
  - build          → docs/index.html מתבנית
  - git push       → GitHub Pages

## כלל קריטי
scored_jobs.json הוא מקור האמת היחיד. משרות שאני מוסיף ידנית מקבלות שדה
initial_status והפייפליין לעולם לא מוחק אותן. combined = pipeline + manual.

## פרומפט חיפוש (Opus)
"Find CURRENTLY OPEN jobs (posted last 30 days) for: {roles}.
Each URL must be a DIRECT link to a specific posting. Search-result pages FORBIDDEN.
Return 8-15 as JSON: {company,title,location,url,posted,description}"

## פרומפט ניקוד (Haiku)
"Evaluate this job for [profile]. Return JSON only:
{fit_score 1-10, score_reason, ai_opener, location_ok}. If location not OK → 0."

## אוטומציה
GitHub Actions: cron 3x ביום (ראשון-חמישי), permissions: contents: write.
workflow שני: deploy docs/ ל-GitHub Pages.

## טראקר (docs/index.html) — Material 3, RTL, mobile-first, PWA
HTML יחיד, כל הסטייט ב-localStorage. לכל משרה כרטיס עם:
ציון, סטטוס (שמור/הוגש/ראיון/הצעה/נדחה), סבבי ראיונות, הערות.
כפתורים שמייצרים פרומפט מוכן ל-clipboard: מכתב מקדים, הכנה לראיון, דוסייר חברה.
כפתור "לא רלוונטי — למד מזה" ששומר ל-feedback.json שהסורק קורא בסריקה הבאה.
מסך אנליטיקה: גרף המרה הגשה→ראיון→הצעה. מצב כהה + כפתור החלפה.

## עלויות — השתמש ב-Haiku לניקוד (זול) ו-Opus רק לחיפוש (איכותי).

קרא לי לפני כל החלטת ארכיטקטורה משמעותית. תתחיל באונבורדינג.
```

### צעד 3: מוסיפים את ה-API key
ב-GitHub → Settings → Secrets → Actions → `ANTHROPIC_API_KEY`.

### צעד 4: מפעילים GitHub Pages
Settings → Pages → Branch: main → Folder: /docs. וזהו — יש לך טראקר חי בכתובת קבועה.

## הערה על LinkedIn (חשוב)
אין API ציבורי למשרות LinkedIn מאז 2023, ו-scraping מנוגד ל-ToS. הפתרון החוקי שעובד: **חיפוש ציבורי** (Claude מוצא לינקים ישירים) + **ATS APIs** של Greenhouse/Lever/Ashby (שם רוב חברות ההייטק מפרסמות בלאו הכי).

## מה למדתי

**1. הפרדת מודלים = חיסכון.** Opus לחיפוש (איכות), Haiku לניקוד של עשרות משרות (מהירות וזול). חוסך ~80%.

**2. מקור אמת יחיד.** קובץ JSON אחד שהפייפליין והטראקר קוראים ממנו. ברגע שהפרתי את הכלל — דאטה נמחק.

**3. מערכת AI נבנית מחדש אחרי שהיא קורסת.** הגרסה הראשונה שלי הראתה 14 משרות במקום 52. תיקנתי את באג השורש, הוספתי שכבת הגנה, וזה מה שעובד היום.
